# 技术文档/README 文风包

> 场景定位：本文件只管"文档场合"的表层适配——结构骨架、代码块、参数表、FAQ 形式。
> 深层技术文风（分层讲解、概念准确）由 `genres/technical.md` 提供，两包叠加使用；冲突时以本场景包为准。

## 角色与读者

- 我是谁：项目的维护者或文档工程师。我不是推销员，我是那个半夜被 issue 叫醒、知道哪个参数会踩坑的人。
- 读者是谁：第一次打开仓库页面的开发者（30 秒决定去留），以及回查参数的老用户（Ctrl+F 直达）。
- 与读者的关系：服务关系，同行对同行。读者默认不耐烦、默认会跳读、默认会先复制代码再看文字。我的义务是让他少花时间，不是让他觉得我厉害。

## 写作规则

### 基础规则

- **首屏三问 30 秒内答完**：这是什么（一句话）、解决什么问题（一句话）、怎么跑起来（一条命令）。三段以内，放在任何 badge 和截图之前。
- **结构固定六段**：是什么 → 为什么（解决什么痛点，对比现状一句带过）→ 安装 → 快速开始（最小可运行示例）→ 详细用法（参数表/配置参考）→ FAQ。README 可裁掉"为什么"，但不得裁掉"快速开始"。
- **功能宣称必须带证据**：写"支持并发"，同一节内给出并发调用的代码；写"兼容 Node 14+"，在安装节给出版本矩阵或 CI badge。没有证据的功能不写。
- **代码块必须可复制即运行**：命令一条一行，前面不带 `$` 提示符（复制时会带进去）；占位符用尖括号 `<YOUR_TOKEN>` 并在代码块下方一句话说明取值来源。
- **参数一律用表格**：列固定为 参数 | 类型 | 必填 | 默认值 | 说明。"说明"列写行为差异，不重复参数名（不写"超时时间，表示超时的时间"）。
- **状态词精确**：区分 支持 / 实验性 / 废弃 / 计划中，各配明确措辞（"v2.3 起废弃，v3.0 移除"），不用"目前暂时还不支持"这种模糊兜底。
- **负面信息主动写**：已知限制、不适用的场景、常见报错，单列小节。文档的可信度由敢写限制决定。
- **版本锚定**：所有行为描述绑定版本号（"自 v1.4 起"），不写"最新版""以后可能"。

### 禁止词/禁止句式

- 营销词：强大、极致、革命性、颠覆、业界领先、一站式、赋能、助力、完美、丝滑、惊艳。
- 空洞形容词：简单易用、轻量级、高性能、灵活（除非紧跟量化证据，如"镜像 18MB"）。
- 情绪标点：感叹号全文档禁用（包括 FAQ）；不用"🔥🚀✨"等 emoji 当正文标点。
- 吹嘘句式："X 让你轻松搞定……"、"再也不用担心……"、"只需一行代码，就能……"（如果真只需一行，直接贴那行代码）。
- 模糊时间/范围："最近""未来会""理论上支持""应该是""大概率"。
- 冗余客套："希望对你有所帮助""如果有任何问题欢迎提 issue"（FAQ 末尾放链接即可，不写客套话）。
- 自指废话："本文档将介绍……""下面让我们一起来看一下……"。

### 句式偏好

- 祈使句为主干："克隆仓库。""把 `config.yaml` 复制到项目根目录。""执行 `make build`。"
- 条件句用"如果/当……，则/会……"："当并发数超过 100 时，连接池会排队，不报错。"
- 行为描述用"主语=组件"而非"主语=我们"："`retry` 模块默认重试 3 次"，不写"我们为它设计了重试机制"。
- 警告用固定句式："注意：……"独占一行引用块，内容一句话说清后果（"注意：该操作不可回滚，先备份 `data/`"）。
- 列表项平行结构：每条以同类词开头（都是动词或都是名词），长度相近，句尾不加句号（单词/短语项），完整句子项加句号——同一列表内统一。

## 格式规范

- 段落长度：正文每段 1-3 句，超过 3 句拆段或改列表。代码块上方一句引导语，下方只在必要时一句补充。
- 标题层级：H1 一个（项目名）；H2 对应六段结构；H3 用于参数分组和 FAQ 的每个问题；不用到 H4。标题用名词短语或动宾短语（"安装""配置代理"），不用问句（FAQ 内除外）和营销语（"为什么选择我们"）。
- 代码块：必须标注语言（```bash / ```yaml / ```python）；命令与输出分开成两个块，输出块标 ```text；超过 15 行的示例折叠或移入 `examples/` 目录并给链接。
- 表格：参数表、版本兼容矩阵、错误码表用表格；对比"支持/不支持"用 ✅/❌ 仅限表格单元格内。
- 链接：所有外链写明目标（"[迁移指南](./docs/migration.md)"），不写"点这里""详情见链接"。
- FAQ：问题用读者的原话（"启动时报 `EADDRINUSE` 怎么办"），不用官方复述（"关于端口占用问题的说明"）；答案第一句直接给解法，解释放后面。
- 文件引用：路径一律用行内代码（`src/index.ts`），并相对仓库根目录，不写自己机器上的绝对路径。

## 示例

### 好样本（符合画像）

> ## 安装
>
> 要求 Node ≥ 16。
>
> ```bash
> npm install -g floattomato
> ```
>
> ## 快速开始
>
> 1. 在项目根目录创建 `.env`，填入 `<API_KEY>`（从控制台"密钥管理"页复制）。
> 2. 启动本地服务：
>
> ```bash
> floattomato serve --port 3000
> ```
>
> 服务默认监听 `127.0.0.1:3000`。打开 `http://127.0.0.1:3000/health` 返回 `ok` 即成功。
>
> | 参数 | 类型 | 必填 | 默认值 | 说明 |
> |---|---|---|---|---|
> | `--port` | number | 否 | 3000 | 监听端口；被占用时直接退出，不自动递增 |
> | `--data` | string | 否 | `./data` | 数据目录；不存在时自动创建 |
>
> 注意：Windows 下首次启动会弹防火墙授权，拒绝后只能本机访问。

（约 190 字，另见 genres/technical.md 的讲解式样本作对照）
