# 邮件/职场沟通文风包

> **定位：本文件是场景包，只管传播场合的表层适配**——主题行、称呼、开头结尾、
> 行动项呈现、抄送与附件。内容的深层文风（专业密度、论证方式、叙事与否）
> 由 `genres/` 下的体裁包决定。写作时与对应体裁叠加加载，规则冲突时本包优先。

---

## 角色与读者

- **我是谁**：组织中的一名协作者——可能是下属、平级、跨部门接口人，或对外的乙方。
- **读者是谁**：忙碌的同事、上级或客户。他在收件箱列表里扫读，一封邮件平均停留不超过 30 秒，往往在手机上竖屏阅读。
- **与读者的关系**：事务协作关系。同组熟人语气可略松，跨部门、客户、高层语气略紧；但无论亲疏，"行动导向"这一条不变。
- **场景本质**：邮件是一张书面任务委托单——可转发、可归档、可追溯，落笔即留痕。写邮件不是在聊天，是在向对方的任务队列里投递一条可执行、可回查的指令。因此：承诺要写清时限，责任要写清主体，口头说过的共识要落成文字。

## 维度画像

| 维度 | 分值 | 说明 |
|---|---|---|
| concise_elaborate | 8 | 扫读场景，一封邮件只说一件事。信息密度高，垫话零容忍，正文争取 150 字内 |
| formal_casual | 6 | 比公文松、比聊天紧：不用"此致敬礼"的公文程式，也不用表情符号、网络梗、语气词堆叠 |
| reserved_expressive | 3 | 情绪让位于事实。催办、拒绝、追责都用陈述句+客观后果表达，不用感叹号和反问句施压 |
| plain_imagery | 2 | 零比喻零意象。"时间紧"就写"截止周三 17:00"，不写"时间像沙漏在流逝" |
| abstract_concrete | 7 | 行动项必须落到人、事、时间三要素。不写"尽快推进""有空看一下"这类空指代 |

> 未列维度默认 5。`formal_casual=6` 虽偏离中性不大，但它是本场景的立身线：
> 滑向 8 就成了公文（程式压过行动），滑向 4 就成了聊天（随意损害留痕效力），必须守住。

## 写作规则

### 基础规则

**结构规则**

1. 一封邮件只办一件事。两件事写两封——便于对方分别回复、转发、归档，也避免一件事卡住另一件。
2. 主题行 = 动作 + 对象 + 时限，全行不超过 20 字。对方在列表页就该知道要不要立刻点开。
3. 首段第一句说清"这封邮件需要你做什么"，背景压到第二句之后，能删则删。
4. 行动项单独成段或列点，每条含三要素：**谁、做什么、何时前**。三要素缺一，重写这条。
5. 需要对方决策时给选项不给开放题："按方案 A 还是方案 B 执行"，不写"您看怎么办"。
6. 给默认路径降低回复成本："周四 17:00 前未收到回复，按方案 A 推进。"让对方连"同意"两个字都可以省。
7. 结尾只留一句礼貌收尾，一个收尾动作对应一件事，不叠加第二层请求。

**语气规则**

8. 对上级、客户、外部用"您"，平级用"你"，全篇统一，不混用。
9. 催办和追责用"事实 + 时限 + 影响"三段式："名单还差 3 个部门未回。今天 17:00 截止，逾期发布会邀请顺延。"不质问、不阴阳。
10. 拒绝用"结论 + 一句原因 + 替代方案"，不铺垫情绪、不连环道歉。
11. 收到请求先确认时限再谈困难："收到，周三给初稿。数据缺口我今晚补，若有风险周二前告知。"
12. 书面留痕意识：口头已达成的一致，邮件里复述一遍并写"如无异议按此执行"；自己造成的延误，写"我漏了确认环节"，不写"流程上有些问题"。

### 禁止词/禁止句式

**禁止词**

- "百忙之中""拨冗"——恭维是阅读负担，直接说事。
- "叨扰""冒昧打扰""不好意思占用您的时间"——卑微开场只拖延行动，删掉后第一句必须是来意。
- "尽快""asap""有空的时候""方便的话"——模糊时间词，一律换成具体日期+钟点。
- "大概""可能""也许""差不多""应该吧"——不确定副词。要么确认后写死，要么写明"待确认，周三回复准确数"。
- "我觉得""我感觉"——个人化模糊判断，换成依据："根据排期表""数据显示"。
- 感叹号、"~"、emoji、网络用语（"宝子""绝绝子""栓 Q"）——职场邮件情绪载体只有标点里的句号。
- "现将……汇报如下""特此通知""妥否，请批示"——公文腔。邮件不是公文，公文场景由 `genres/gongwen.md` 管，此处不越界。
- 签名档里的名人名言、格言警句。

**禁止句式**

- 环形开场："好久没联系了，最近怎么样？……其实有件事想请你帮忙。"——禁止寒暄前置，第一句必须说事。
- 推卸式被动："这个问题没有被及时处理。"——必须写明责任主体："我漏了确认环节，今天补回。"
- 反问施压："难道你没看到我上周五的邮件吗？"——换成事实+时限："周五邮件的方案待确认，今天 17:00 前给我结论即可。"
- 连环道歉："非常抱歉实在不好意思耽误了您宝贵的时间……"——道歉限一次，后半句必须是补救方案。
- 长定语堆叠："关于上周我们讨论过的那个需要市场部配合的新版落地页的设计稿"——拆成短句："上周定的落地页设计稿（需市场部配合），初稿见附件。"

### 句式偏好

- **主题行骨架**：`[动作] + [对象] + [时限]`
  - 「请确认：发布会嘉宾名单（周四 17:00 前）」「请审批：Q3 预算表 v2（周三前）」「FYI：8 月考勤已归档，无需回复」
- **开头句骨架**：直接陈述来意。
  - "这封邮件想确认 X 的排序。""同步一下 X 的进度：已完成 80%。"
- **行动项骨架**：`请 + [人] + 在 [时限] 前 + [动作] + [交付物]`
  - "请市场部在周三 12:00 前把终版 logo 发到共享盘。"
- **进度同步骨架**：结论先行 + 风险一句 + 对策一句。
  - "X 已完成 80%，预计周五交付。风险：数据源未到位。对策：已联系 IT，今晚补齐，缺口不影响主流程。"
- **拒绝骨架**：结论 + 一句原因 + 替代方案。
  - "这次接不了，排期已满到月底。两个替代：a) 下月初启动；b) 我推荐小李，他做过同类项目。"
- **催办骨架**：事实 + 截止时间 + 客观影响。
  - "名单还差 3 个部门未回。今天 17:00 截止，逾期发布会邀请顺延一天。"
- **收尾骨架**（三选一，不叠加）：
  - "有问题随时找我。" / "期待你的反馈。" / "如无异议，我按此推进。"

## 格式规范

- **段落长度**：每段不超过 3 行；正文总量建议 150 字以内。超过 300 字先自检：是否该拆成两封，还是细节该挪进附件。
- **段落结构**：3-5 段为宜——来意段 → 背景段（可省，限一句）→ 行动项段 → 收尾段。
- **列表**：行动项 ≥2 条必须用数字列表，每条一行、动词开头；不要写成一坨逗号分隔的长句。
- **称呼**：单独一行顶格。"张老师，您好："或"Hi 李明，"。群发用"各位同事，大家好："。
- **结尾敬语**：用"谢谢！"或"Best regards,"即可；不用"此致 敬礼"（公文程式）。
- **签名**：姓名 / 部门 / 电话，三行以内。
- **附件**：正文必须点名附件及用途，并指路重点："附件是 Q3 预算表 v2，改动在第 3 页，已标黄。"
- **抄送**：需要谁知情才抄谁，正文里交代一句"抄送王经理知悉"；不搞抄送轰炸，不用密送做职场博弈。
- **回复**：同一事项留在原线程里回复，不另起新主题；回复时先答行动项，再补充说明。
- **加粗**：只对时限和关键动作加粗，一封邮件加粗不超过 3 处；全段加粗等于没有加粗。
